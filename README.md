# AI Knowledge Repository & Custom Generator

このリポジトリは、TBMが提供する「生成AI研修プログラム（基礎編）」の原本データを管理しつつ、**特定の顧客・案件に向けた「カスタマイズ研修」や「専用プロンプト」を、生成AIを使って半自動で作成するためのシステム**です。

## 📁 ディレクトリ（フォルダ）構成：コアと派生の完全分離

リポジトリは、**「絶対に書き換えてはいけない原本（マスター）」**と、**「顧客ごとに生成するカスタマイズデータ」**の役割で分割されています。

- **`01_tbm_base/`** 【マスターデータ】
  - `00_original/`: TBM研修基礎編（全9回）のオリジナルPDF
  - `10_text/`: AIに読み込ませるための、全9回分のテキスト・コンテキスト
  - `20_prompts/`: 全ての基礎となる汎用プロンプト・テンプレート群
    > ⚠️ **注意**: このフォルダ内のファイルは、全顧客共通の「正解データ」として保護されます。直接書き換えないでください。

- **`02_custom_projects/`** 【カスタマイズ出力先】
  - `_template/`: 新規案件のひな形。ここに要件定義書（`client_context.md`）があります。
  - `(例) Client_A_6days/`: ジェネレーターによって生成された、A社向けの全6回のテキストやプロンプトの格納場所。

- **`03_generators/`** 【カスタマイズ自動化プロンプト】
  - 人間が手作業でカスタマイズするのではなく、**AIにカスタマイズ作業をやらせるための指示書**が入っています。


---

## 🚀 カスタマイズ研修の自動作成フロー（ジェネレーターの使い方）

「基礎編（全9回）」の内容を、顧客に合わせて「6回に圧縮し、事例を変更して、プロンプトも専用のものにする」手順です。

### Step 1: 案件フォルダの準備と要件定義
1. `02_custom_projects/_template` フォルダをコピーし、案件名（例：`Client_A`）にリネームします。
2. フォルダ内にある `client_context.md` を開き、「何回に圧縮するか」「どんな事例（業種や業務）を入れるか」などの要件を記入します。

### Step 2: ジェネレーターの実行（AIへの作業指示）
ChatGPTやClaudeなどのAIに、以下の2つを読み込ませます。
- **マスターデータ**: `01_tbm_base/` 内の必要なテキスト（または全体インデックス）
- **顧客要件**: Step1で作成した `client_context.md` 

その上で、`03_generators/` にある**生成用プロンプト**を実行します。

- **カリキュラムを再構築（圧縮・事例変更）したい場合**:
  👉 `01_curriculum_builder.md` をAIに入力してください。
- **プロンプトの内容だけを顧客用に書き換えたい場合**:
  👉 `02_prompt_customizer.md` をAIに入力してください。

### Step 3: 出力結果の保存・調整
AIが出力した新しいMarkdownテキストを、`Client_A` フォルダ内の `10_custom_text` や `20_custom_prompts` に保存します。
必要に応じて、AIと壁打ちを行って微調整を行えば**カスタマイズ研修資料の完成**です！
